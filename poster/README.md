# poster/

Final Milestone 3 poster lives here.

## What to commit

- **`poster.pdf`** — the final 24"×36" poster as a PDF. This is the file the teaching staff grades and that you print.
- **Source file** — the editable version in whatever tool you used, e.g., `poster.pptx`, `poster.key`, `poster.fig` (Figma export), `poster.tex`, `poster.afpub`, etc. If your source is in a cloud tool (Figma, Canva, Google Slides), commit a link in `SOURCE.md` instead of a file.
- **Figures** — put reusable figures (architecture diagram, F1 table, confusion matrices) in `poster/figures/` if they're too large or complex to inline. These can also live in `evals/results/` — pick one location and don't duplicate.

## Required poster sections (per Milestone 3 spec)

1. **Problem Description**
2. **Policy Language** (human-abuser teams) **or Model Safety Spec** (AI-as-abuser teams) — under 400 words, plain language
3. **Technical Back-end**
4. **Evaluation** — F1/precision/recall table + any confusion matrices or plots
5. **Looking Forward**

Plus: **team number in a large font** somewhere prominent, so guest judges can find their assigned posters.

## Printing

Takes ~3 business days at the [Stanford Hub](https://thehub.stanford.edu/services/print-poster), so plan to submit by **Monday, June 1**. [BlockPoster](https://www.blockposters.com/) is a free tiling alternative if you want to print A4 pages and tape them together.

## Design tips

- Less text is better. The poster supports conversation, not replaces it.
- Big headlines, readable from 3 feet away.
- Use Stanford Cardinal (`#8C1515`) sparingly for headings to stay on-brand without overdoing it.
- Include at least one screenshot of your deployed product UI and one of your moderator UI.
- [Stanford Undergrad Research poster tips](https://undergradresearch.stanford.edu/share/surps-asurps/make-good-poster)
